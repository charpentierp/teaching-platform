Why network security matters 5.2.1

Any device connected to a network is potentially accessible from anywhere in the world. This makes networks valuable targets for criminals. Network security is about protecting data, devices, and services from unauthorised access, damage, or disruption.

Three fundamental security goals underpin everything in this topic: keeping data confidential (only authorised users can access it), maintaining its integrity (data is not altered without authorisation), and ensuring availability (systems remain accessible when needed).

Core protection methods

Access control

Ensuring only authorised users can access systems. Includes usernames and passwords, biometric authentication (fingerprints, facial recognition), and multi-factor authentication combining two or more methods.

Physical security

Protecting the physical hardware — locked server rooms, badge access, security cameras. An attacker with physical access to a machine can bypass almost all software security measures.

Firewalls

Hardware or software that monitors incoming and outgoing network traffic and blocks anything that does not meet defined security rules. Acts as a barrier between trusted and untrusted networks.

Encryption

Converting data into a coded form so it cannot be read if intercepted. HTTPS uses encryption (TLS) to protect data in transit. Essential for any sensitive information sent over a network.

Identifying vulnerabilities and protecting systems 5.2.4 / 5.2.5

Finding weaknesses before attackers do

Organisations proactively test their own systems to identify security gaps. The key methods Pearson expects you to know:

Penetration testing

Authorised security professionals attempt to break into a system using the same techniques a real attacker would use. The goal is to find vulnerabilities before malicious actors do, then fix them.

Ethical hacking

Similar to penetration testing but broader — ethical hackers have permission to probe systems, networks, and applications. They report all findings to the organisation rather than exploiting them.

Protecting software systems

Method What it does
Secure coding practices Writing code that validates inputs, handles errors correctly, and avoids common vulnerabilities such as SQL injection or buffer overflow.
Code reviews Having other developers examine code for security flaws before it is deployed. A second pair of eyes catches vulnerabilities the original developer missed.
Modular testing Testing individual components of a system in isolation to verify each module behaves securely before integration.
Audit trails Keeping logs of who accessed what and when. If a breach occurs, audit trails allow investigators to trace what happened and identify the attacker.
Securing the OS Keeping the operating system updated with security patches, removing unnecessary services, and configuring user permissions carefully.

Cloud security note: storing data in the cloud introduces specific risks — data is held on third-party servers, meaning the organisation has less direct control over physical security. Data breaches at the provider level can expose customer data even if the organisation's own systems are secure.

The Internet and the World Wide Web 5.3.1 / 5.3.2

Critical distinction — Paper 1 trap: The internet and the World Wide Web are NOT the same thing. The internet is the global network of interconnected computers. The WWW is a collection of web pages and resources accessed via the internet using HTTP/HTTPS. The WWW runs on top of the internet — email, FTP, and online gaming also use the internet but are not part of the WWW.

Click each component to understand its role, how it works, and what Pearson expects you to know about it.

What it is
Every device connected to the internet is assigned a unique IP address. This address allows data packets to be routed to the correct destination across the network.
IPv4
32-bit address written as four groups of numbers (e.g. 192.168.1.1). Supports approximately 4.3 billion unique addresses — no longer sufficient for the number of devices online today.
IPv6
128-bit address written in hexadecimal (e.g. 2001:0db8:85a3:0000:0000:8a2e:0370:7334). Supports an effectively unlimited number of addresses — introduced because IPv4 addresses ran out.
Paper 1
Know that IPv6 was introduced because IPv4 ran out of addresses. Know that IPv4 is 32-bit and IPv6 is 128-bit. Be able to identify an IPv6 address from its hexadecimal format.
Purpose
Humans use domain names (www.google.com) because they are memorable. Computers communicate using IP addresses. DNS translates between the two — it is effectively a phone book for the internet.
How it works
When you type a URL, your device sends a request to a DNS server asking for the IP address of that domain. The DNS server responds with the IP address, and your device connects to that address to load the page.
Paper 1
"Describe the role of a DNS server when a user types a URL into a browser" is a classic 2-mark question. You must describe both steps: the request from device to DNS server, and the response with the IP address.
Components
The WWW consists of web servers (computers that host web pages), URLs (the addresses of those pages), HTTP/HTTPS (protocols used to request and deliver pages), and HTML (the language web pages are written in).
Web server
A computer that stores web pages and delivers them to browsers on request. When you navigate to a URL, your browser sends an HTTP/HTTPS request to the web server, which responds with the page content.
ISP
Internet Service Provider — the company that provides your internet connection. Your ISP routes your traffic between your home network and the wider internet.
Paper 1
Know the difference between the internet and the WWW. Know the role of web servers, URLs, HTTP/HTTPS, and HTML as components of the WWW. ISP may appear in "state the role of" questions.
Modem
Converts signals from your ISP (analogue or digital) into a form your home network can use. Connects your network to the internet via the ISP infrastructure.
Router
Directs data packets between different networks — typically between your home LAN and the internet. Assigns IP addresses to devices on your local network (DHCP) and manages traffic in and out.
Switch
Connects multiple wired devices within a LAN and directs data only to the specific device it is addressed to — more efficient and secure than a hub, which broadcasts to all devices.
WAP
Wireless Access Point — allows wireless devices to connect to a wired network. Often combined with a router in home networking equipment. Extends the network wirelessly.
Paper 1
Know the distinct role of each device. A common question gives a scenario and asks which device is responsible for a specific function. Modem = connect to ISP. Router = direct traffic between networks. Switch = connect devices within a LAN. WAP = wireless connectivity.
